# Rubber Duck Mode (opt-in)

Before implementing any non-trivial change:
- Explain your approach in 3-5 sentences of plain English
- State: what you'll change, why, and what could go wrong
- Wait for the user to say "go" before writing any code

Skip this for:
- One-line fixes, typos, formatting
- Tasks where the user already described the exact implementation
- Follow-up changes in an approved plan
