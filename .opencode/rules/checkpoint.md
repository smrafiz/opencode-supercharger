# Checkpoint Mode (opt-in)

When working through a multi-step plan, checklist, or phased implementation:
- Complete one step at a time
- After each step, pause and ask: "Step N done. Ready for the next step, or want to review first?"
- Do not proceed to the next step until the user confirms
- If the user says "continue" or "go", proceed without pausing until told otherwise
- If the user says "review", present what was done and wait for feedback

This does NOT apply to:
- Simple single-step tasks
- Tasks where the user said "do it all" or "proceed"
- Bug fixes, typo corrections, or trivial changes
